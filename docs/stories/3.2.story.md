# Story 3.2: Configuration des Coefficients par Niveau et Série

## Status: Ready for Review

## Story

**As an** Admin,
**I want** définir le coefficient de chaque matière pour chaque niveau et série,
**so that** le système puisse calculer correctement les moyennes pondérées.

## Acceptance Criteria

1. Un écran "Configuration des Coefficients" affiche un sélecteur de niveau et de série
2. Après sélection d'un niveau/série, un tableau affiche les matières configurées avec leurs coefficients :
   - Colonnes : Matière (code + nom), Coefficient, Heures hebdomadaires, Actions
   - Ligne de total : Somme des coefficients, Somme des heures hebdomadaires
3. Je peux ajouter une matière à ce niveau/série via "Ajouter une matière" :
   - Select matière (uniquement les matières actives non encore ajoutées à ce niveau/série)
   - Coefficient (numérique, 1 à 8)
   - Heures hebdomadaires (numérique, optionnel)
4. Je peux modifier le coefficient ou les heures d'une matière existante
5. Je peux supprimer une matière de ce niveau/série
6. La suppression est bloquée si des notes existent pour cette matière/niveau/série
7. Le système empêche la création de doublons (même matière pour le même niveau/série)
8. Pour les niveaux 6e à 2nde, la série est "Tronc commun" (pas de sélection de série)
9. Pour les niveaux 1ère et Tle, la série est obligatoire (A, C, ou D)
10. Des coefficients par défaut sont pré-configurés lors de l'initialisation du tenant (seeder)

## Dev Notes

### Data Model [Source: docs/architecture/data-models.md#subject-class-coefficients]
```sql
CREATE TABLE subject_class_coefficients (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    subject_id BIGINT UNSIGNED NOT NULL,
    level_id BIGINT UNSIGNED NOT NULL,
    series_id BIGINT UNSIGNED NULL,
    coefficient DECIMAL(3,1) NOT NULL,
    hours_per_week TINYINT UNSIGNED NULL,
    created_at, updated_at,
    UNIQUE KEY (subject_id, level_id, series_id),
    FK subject_id -> subjects(id) ON DELETE CASCADE,
    FK level_id -> levels(id) ON DELETE CASCADE,
    FK series_id -> series(id) ON DELETE SET NULL
);
```

### Validation Série Conditionnelle
Pour les niveaux 1ERE et TLE, la série est obligatoire. Logique:
```php
public function rules(): array
{
    $level = Level::find($this->level_id);
    $seriesRequired = $level && in_array($level->code, ['1ERE', 'TLE']);

    return [
        'series_id' => [$seriesRequired ? 'required' : 'nullable', 'exists:series,id'],
        // ...
    ];
}
```
