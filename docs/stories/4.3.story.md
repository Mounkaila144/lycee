# Story 4.3: Vérification de la Couverture des Matières

## Status: Aproved

## Story

**As an** Admin,
**I want** voir quelles matières ne sont pas encore affectées à un enseignant dans chaque classe,
**so that** je puisse m'assurer que toutes les matières sont couvertes avant le début du semestre.

## Acceptance Criteria

1. Un écran "Vérification de couverture" affiche pour chaque classe : liste des matières avec coefficient et enseignant affecté ou badge "Non affecté"
2. Le tableau est filtré par cycle, niveau, série
3. Un pourcentage de couverture est affiché par classe (ex: "8/10 matières couvertes - 80%")
4. Un pourcentage de couverture global est affiché
5. Un bouton "Affecter" est disponible sur les lignes "Non affecté"
6. Les classes avec couverture < 100% sont mises en évidence

## Tasks / Subtasks

- [ ] Task 1: Créer l'endpoint de couverture (AC: 1-6)
  - [ ] `GET /api/admin/teacher-assignments/coverage?academic_year_id=X&cycle_id=Y&level_id=Z`
  - [ ] Pour chaque classe: lister les matières configurées (via coefficients) et indiquer si un enseignant est affecté
  - [ ] Calculer le pourcentage de couverture par classe et global

- [ ] Task 2: Structurer la réponse (AC: 1, 3, 4)
  - [ ] Format: `{ classes: [{ id, name, level, series, subjects: [{ id, code, name, teacher: {name} | null }], coverage_percent }], global_coverage_percent }`

- [ ] Task 3: Tests (AC: 1-6)
  - [ ] Test couverture 100%
  - [ ] Test couverture partielle
  - [ ] Test classe sans aucune affectation

### Logique de Couverture
Pour chaque classe de l'année active:
1. Trouver le level_id et series_id de la classe
2. Récupérer les subject_ids des coefficients pour ce level/series
3. Vérifier lesquels ont un TeacherSubjectAssignment pour cette classe
4. Calculer: couverture = assigned_count / total_subjects * 100
