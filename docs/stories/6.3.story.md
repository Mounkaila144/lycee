# Story 6.3: Rapport de Validation de la Structure

## Status: Aproved

## Story

**As an** Admin,
**I want** générer un rapport de validation de la structure académique,
**so that** je puisse identifier les incohérences avant le début du semestre.

## Acceptance Criteria

1. Un bouton "Générer rapport de validation" est disponible sur le dashboard
2. Le rapport affiche les catégories suivantes avec indicateurs (vert/orange/rouge) :
   - Classes sans professeur principal
   - Niveaux/Séries sans coefficients
   - Matières sans enseignant affecté
   - Classes en sureffectif
   - Classes vides
   - Cohérence des barèmes
   - Enseignants sans affectation
3. Chaque section affiche un compteur
4. Un score global de complétude est affiché (ex: "Structure complète à 87%")
5. Le rapport est exportable en PDF
6. Chaque ligne du rapport a un lien "Corriger"

## Tasks / Subtasks

- [ ] Task 1: Créer l'endpoint rapport de validation (AC: 1-4)
  - [ ] `GET /api/admin/structure/validation-report?academic_year_id=X`
  - [ ] Calculer chaque catégorie de validation

- [ ] Task 2: Implémenter les vérifications (AC: 2, 3)
  - [ ] classes_without_pp: Classes sans head_teacher_id
  - [ ] levels_without_coefficients: Couples (level, series) actifs sans coefficients

  - [ ] overcrowded_classes: enrolled_count > max_capacity

- [ ] Task 4: Export PDF (AC: 5)
  - [ ] `GET /api/admin/structure/validation-report/export?format=pdf`
  - [ ] Template Blade pour le rapport

- [ ] Task 5: Tests (AC: 1-6)
