# Story 4.1: Affectation d'un Enseignant à une Matière et des Classes

## Status: Aproved

## Story

**As an** Admin,
**I want** affecter un enseignant à une matière dans une ou plusieurs classes,
**so that** l'enseignant puisse saisir les notes de ses élèves dans ces classes.

## Acceptance Criteria

1. Un écran "Affectations Enseignants" affiche la liste des affectations existantes pour l'année active
2. Un bouton "Affecter un enseignant" ouvre un formulaire avec :
   - Enseignant (select avec recherche : liste des utilisateurs avec rôle "Enseignant")
   - Matière (select : liste des matières actives)
   - Classe(s) (multi-select : classes où cette matière a un coefficient configuré)
   - Année scolaire (select, défaut : année active)
3. La sélection de la matière filtre automatiquement les classes disponibles
4. Le multi-select de classes permet d'affecter l'enseignant à plusieurs classes en une seule opération
5. Le système vérifie qu'aucune autre affectation n'existe déjà pour la même matière/classe
6. Si une affectation existe déjà, un message d'erreur s'affiche
7. Un message de succès s'affiche après création
8. La charge horaire résultante est affichée

## Dev Notes

### Data Model [Source: docs/architecture/data-models.md#teacher-subject-assignments]
```sql
CREATE TABLE teacher_subject_assignments (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    teacher_id BIGINT UNSIGNED NOT NULL,
    subject_id BIGINT UNSIGNED NOT NULL,
    class_id BIGINT UNSIGNED NOT NULL,
    academic_year_id BIGINT UNSIGNED NOT NULL,
    created_at, updated_at,
    UNIQUE KEY (teacher_id, subject_id, class_id, academic_year_id),
    INDEX (teacher_id, academic_year_id),
    INDEX (class_id, academic_year_id)
);
```
**Note PRD**: La contrainte unique devrait être sur (subject_id, class_id, academic_year_id) - une matière par classe n'a qu'un seul enseignant.

### Affectation en Masse
Le store() accepte un array de class_ids et crée plusieurs affectations en une fois:
```php
public function store(StoreTeacherAssignmentRequest $request): JsonResponse
{
    $created = [];
    foreach ($request->class_ids as $classId) {
        $created[] = TeacherSubjectAssignment::create([
            'teacher_id' => $request->teacher_id,
            'subject_id' => $request->subject_id,
            'class_id' => $classId,
            'academic_year_id' => $request->academic_year_id ?? AcademicYear::active()->first()->id,
        ]);
    }
    // ...
}
```

### Charge Horaire [Source: docs/prd/module-structure-academique.md#FR44]
La charge horaire est calculée en joignant les coefficients:
```php
// Sur le model ou resource
$weeklyHours = SubjectClassCoefficient::where('subject_id', $assignment->subject_id)
    ->where('level_id', $assignment->class->level_id)
    ->where('series_id', $assignment->class->series_id)
    ->value('hours_per_week');
```
